================================================================================
                          TECHNICAL DETAILS
================================================================================

ISSUE ROOT CAUSE:
-----------------
JSX/React uses double quotes for attribute values. When quotes are escaped
with backslashes (e.g., path=\"/example\"), the JSX parser treats them as
literal characters rather than string delimiters, causing syntax errors.

CORRECT SYNTAX:
--------------
<Route path="/example" element={<Component />} />

INCORRECT SYNTAX:
----------------
<Route path=\"/example\" element={<Component />} />

ESBuild ERROR MESSAGE:
---------------------
[vite:esbuild] Transform failed with 1 error:
Expected "{" but found "\\"

SOLUTION:
---------
Remove backslash escaping from JSX attributes. In JSX, quotes within
attribute values are already treated as string delimiters and don't
require escaping.
